Idea: Exploratory Factor Analysis (EFA)

Locating Data

You’ll want to first locate a raw data set with (a) survey responses using the same column and row formatting structure as the set that was used in the example AND (b) a code book that has clearly defined the variables (columns). Make sure you keep an accurate record of what data set was used and where you got it from.

Here’s the same list of possible resources from last week:

  • AWS public datasets - This stands for Amazon Web Service and yes its that Amazon. Believe it or not, Amazon is one of the largest providers of APIs (acronym for application programming interface which is a fancy way to say a service where computer programs can talk to each other), data analytics (fancy of way of saying that a computer is being used for stats), data processing (fancy way of say how efficient data can be analyzed, rearranged, whatever), and storage cloud computing (fancy way of saying securing, sharing and storing data and sharing it). To this end, a lot and I mean a lot of data sets are publicly available.

  • Baidu Research - Baidu provides a wide range of data sets at no cost for research and personal uses. They provide data from the various business units like automated car testing, medical, video highlight. While it is unlikely you’ll locate a survey response set, this is a good thing to note.

  • data.gov - In the United States, all publicly funded data sets are available barring those that may pose a security threat (but you can even get those eventually using the Freedom of Information Act). Other bodies freely share their data as well such as the UK, the EU, and India. There are plenty of other organizations that share their data too. For example, whatever your opinion is on the World Bankor even if you don’t have one, even they shares their data since it is publicly funded as well!

  • data.world - One of the best publicly available repos out there. Some of the sets even come with full R scripts! Other software scripts like Excel, Python, SPSS, SAS, etc are available as well. This is strictly user driven so availability is contingent on who shares what and how.

  • Github - Not only are public data sets plentiful on Github, but people have also taken the time to compile a list for the rest of us. Here are a few popular ones (I promise not every repo is named awesome on Github):

    • Awesome Public Datasets
    • Awesome Data

  • Google Dataset Search - Much like nearly every other Google service (e.g. Google Scholar), Goggle Datasets doesn’t actually house anything, rather it is a means for you to get to data, aka a portal what a fantastic name), without having to search all over the web. It is fairly new and their database grows every second - actually its faster than that but as humans we have trouble compartmentalizing any unit of time smaller than a second. Now if you’ve heard the term indexing, that is not this. The best analogy I can think of is creating a grocery shopping list as needed through the week (aka adding to the data base) and then reordering and grouping food items that are closest in proximity to each other so that you can get out of the store as quick and efficiently as possible1.

  • Kaggle - Described as an AirBnB for data scientists, this site has a ton of public and sometimes private data that has been cleared for public consumption. More than that, it is an absolute playpen if you like exploring data. If you go headfirst into a data set and cannot and I mean cannot cognitively picture an end goal or what the data will tell you until something is done with it, this is the place for you!

What to Turn in

After you have located a data set, do the data scientist thing and please follow these steps:

  1. Open up the codebook and locate multiple columns:
  • one that has a unique identifier
  • one quantiative column of your choice that is not filled with demographical information.

  1. Use the same steps outlined in the example piece to conduct your own exploratory factor analysis (EFA).

  2. Answer the following questions on a word document. As usual, please put your name in the upper left hand corner.

  1. What is the official name of the data set you chose?
  2. What repository did you get it from (e.g. data.world, Kaggle, etc)? If you did not get it from a place where data is housed, please copy and paste the web address. Please do not copy and paste any additional nonsense that say search engines put on a web address. For example, if you’ve ever copied and pasted a website after using Google, you may notice that the address is obnoxiously long. I don’t want that. If you are having trouble, contact me and I’ll help you through it.
  3. How did you determine the number of factors?
  4. What are the factors likely representative of? hint: look at the loadings
  5. Describe the results into a single paragraph narrative. Try to be specific and concise. There is a very narrow way to interpret the results of a factor analysis.
  1. Submit your script, a copy of the data set and responses on eCampus.
